## Your job: Fix the dot-path resolver to support explicit $ prefix

### The Problem
The current resolver in `apps/nexla-service/src/nexla_service/resolver.py` treats any string without `.` or `[` as a literal. This means `extract_value({price: 215.5}, "price")` returns the string `"price"` instead of `215.5`. Flat payloads can NEVER be accessed via path.

### The Fix: `$` prefix convention
Modify `resolve_dot_path()` in resolver.py:
- If the path string starts with `$`, strip the `$` and resolve the remainder as a dot-path (split by `.`, handle `[N]` array indexing as before). 
  - Example: `"$current.temperature_2m"` on `{current: {temperature_2m: 26.7}}` → `26.7`.
  - Example: `"$features[0].properties.mag"` on GeoJSON → `5.2`.
- If the path string does NOT start with `$`, return it as a literal string unchanged.
  - Example: `"gCO2/kWh"` → `"gCO2/kWh"`.
  - Example: `"magnitude"` → `"magnitude"`.
- Remove the `_is_literal()` function or repurpose it — the logic shifts from detecting literals to detecting the `$` prefix.
- `extract_value()` signature stays the same: `extract_value(payload: dict, path: str) -> Any`.
- Update all docstrings.
